The problem it solves

There are two problems here — one on the surface, one underneath. The second is the reason this project exists.

1

The everyday problem

Company research is slow, manual, and repetitive.

To understand a competitor, sales prospect, or investment, someone opens a dozen tabs, skims long pages, copy-pastes the useful bits, organises it into something readable, and tracks every source. That's 30–60 minutes of tedious work per company — and the output is inconsistent every time.

👉 This tool turns one company name into a clean, sourced one-page brief in seconds.

2

The deeper problem (the real point)

Teams over-use autonomous "AI agents" and can't tell when they're the wrong choice.

Agents — AI that decides its own steps — are hyped. But when a task's steps are already known, an agent is often costlier, slower, and less reliable than a simple fixed sequence of steps (a workflow) — yet teams reach for the agent anyway, burning cost and trust.

👉 This project builds the same task both ways, measures them, and shows with real numbers when the simpler workflow wins.

What it is & why it's useful

The tool is real and useful on its own — but the headline deliverable is the decision behind it.

The everyday use

Pick a subject type — a company, a political leader, a political party, or any topic / current event — and type a name (e.g. "Notion"). The tool searches the web for free, reads the pages, and produces a structured, sourced brief in seconds:

📌 Overview 🧩 Products 🎯 Positioning 📰 Recent news ⚠️ Caveats & gaps 🔗 Cited sources

It replaces 30–60 minutes of manual tab-opening, skim-reading, and copy-pasting.

The real point

The same task is built two ways so the comparison is honest, not rhetorical:

WORKFLOW A fixed pipeline — gather → summarise → synthesise → format. Predictable, cheap, debuggable.
AGENT An autonomous loop that picks its own tool calls until it decides it's done. Flexible, but costlier & slower here.

The deliverable is DECISION_DOC.md: it argues why a workflow beats an agent for a task whose steps are known in advance — backed by measured cost, latency, and failure-mode analysis. That's the senior signal: default to the simplest thing that works, and write down the trigger to upgrade.

How to use it

Two ways to run it: a command line for measurement, or a clean web page for demos. Set-up is one-time and free.

1

Install & add a free key

Get a free Groq API key (no credit card) from console.groq.com/keys.

# one-time setup
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt

# paste your key into .env
copy .env.example .env
2

Generate a brief

Run either engine on any company. Output is a clean markdown brief.

# the deterministic workflow
python cli.py run --mode workflow --company "Notion"

# the autonomous agent
python cli.py run --mode agent --company "Notion"
3

Compare them (the proof)

Run both on the same input and get a paste-ready cost/latency table.

python cli.py benchmark --company "Notion"

# → benchmarks/latest.md
#   real tokens, cost, wall-time

Prefer a web page?

A single-page UI (FastAPI + Tailwind) over the same engine — toggle Workflow/Agent and watch cost & latency update live.

uvicorn webapp.app:app --reload
# open http://127.0.0.1:8000

Ships ready to deploy as a public demo on Hugging Face Spaces — visitors get a few free briefs, then bring their own free key. See DEPLOY.md.

Want full reproducibility?

Skip web search and feed it exact URLs — handy when you want a deterministic demo or you already know the sources.

# one line (PowerShell-safe)
python cli.py run --mode workflow --company "Notion" --url https://example.com/about --url https://example.com/news

Any "should this be an agent?" call

The real transferable asset: the framework for when to use a workflow vs an agent applies to every LLM automation decision.

The reusable principle

Most "AI features" don't need an autonomous agent. When the steps are known, a deterministic workflow is cheaper, faster, and safer — and you only reach for an agent when the path itself becomes unknowable. This project is a repeatable template for making — and defending — that call.
